==================================================
PRIMARY GOAL
==================================================

Make Soul Codex profiles feel specific to the individual.

Two users who share the same Sun sign or broad system labels must still get noticeably different profile blurbs if their behavioral patterns, timing, stress style, or signal intersections differ.

The engine must stop producing:
- generic zodiac clichés
- therapist-blog filler
- repeated “you are intuitive / deep / sensitive” sludge
- same-person-different-name profiles

==================================================
PHASE 2 — DISTINCTIVENESS SCORING MODEL
==================================================

Implement scoring for each candidate statement using these dimensions:

1. behavioral_specificity
2. distinguishing_power
3. contradiction_depth
4. phase_relevance
5. action_usefulness
6. language_originality

Use a weighted scoring model.

Suggested weighting:
- behavioral_specificity: 0.25
- distinguishing_power: 0.25
- contradiction_depth: 0.15
- phase_relevance: 0.15
- action_usefulness: 0.10
- language_originality: 0.10

Add penalties for:
- generic adjectives
- zodiac stereotype phrasing
- spiritual filler
- highly reusable/template-like blurbs
- repeating sentence frames too often

Reject statements if:
- final score is below threshold
- they contain banned language
- they contain no real behavior
- they could fit too many users with only trivial edits

==================================================
PHASE 5 — CONTRADICTION-AWARE PROFILE LOGIC
==================================================

Force profiles to sound human by using contradiction.

Each profile should favor statements that capture tension like:
- appears calm but is mentally braced for disappointment
- looks independent but still longs for recognition
- seems flexible but uses control as protection
- gives a lot, then resents silently

At least one major profile paragraph should contain:
- surface pattern
- hidden pattern
- cost of that contradiction

==================================================

Build profile sections around:
1. core_pattern
2. pressure_pattern
3. blind_spot
4. current_phase
5. action_truth

Rules:

  - one modifier
  - one behavior or pressure signal
- timing/current phase must modify tone and advice
- stress/behavior should outrank flattering generic copy
- every section should end with a cost, interruption, or action when appropriate

==================================================

- sound precise, not mystical-mushy
