What I’d tighten before you run it

1. Don’t overcommit to Profile and home hero areas

Your current app seems to use Profile as the front door in practice. If there is no true separate “home hero,” say:

“ProfilePage and any primary snapshot/home hero surface”

That avoids the agent inventing a second hero just because the brief sounded poetic.

2. Keep tailwind.config.ts optional

If your styling is already mostly driven from index.css, don’t force Tailwind changes unless needed.

Change:
	•	tailwind.config.ts

To:
	•	tailwind.config.ts only if needed for token exposure

That prevents unnecessary config churn.

3. Be explicit about nav mode behavior

Right now “mode-toggle remains functional” is good, but I’d add:
	•	STD / ADV must remain readable and intentionally styled, not look like leaked internal labels

Because that’s still one of the weaker bits in the UI.

4. Protect readability on inner pages

Add one rule:
	•	Background watermark/logo motif must not appear behind dense reading sections like Codex body copy or transit lists

Otherwise some agent will slap a glowing sigil behind five paragraphs and call it atmosphere.

5. Don’t let the sidebar become too dominant

Say this explicitly:
	•	Sidebar should feel elegant and quiet, not heavier than the content area

Because 260px is fine, but easy to make visually overbearing.
